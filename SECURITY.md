# Security Policy

## Reporting a Vulnerability

Please do not open public issues for security problems.

Report vulnerabilities privately to the maintainer and include:

1. A description of the issue and impact.
2. Steps to reproduce.
3. Any relevant logs or screenshots with secrets redacted.

You should receive acknowledgement within 3 business days.

## Scope

Security reports are especially valuable for:

1. Authentication and allowlist bypasses.
2. Command execution or sandbox escape risks.
3. Secret exposure in logs, config, or chat outputs.
